# Contributing

We welcome optimizations to the Concise Response Protocol.

## Rules
1.  **Reductionism:** Pull requests must decrease token count or increase clarity.
2.  **No Feature Creep:** Do not add conversational features.
3.  **Format:** Submit changes using the protocol's style (bullet points, no filler).

## Process
- Fork the repo.
- Create a branch.
- Submit a PR with a description < 50 words.
